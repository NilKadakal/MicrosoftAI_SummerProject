Project overview

Built as a complete local RAG workflow.

This project is not only a chatbot interface. It includes document ingestion, embeddings, local database storage, context retrieval, guarded answering, automated tests, and a presentation website.

Local models

Uses Foundry Local models for embeddings and chat generation on the user's machine.

Document Q&A

Supports TXT, PDF, and DOCX files for local document-based question answering.

SQLite storage

Stores chunks and embeddings in a lightweight local SQLite database.

Grounded answers

Refuses unsupported questions when the answer is not found in the provided documents.

Local LLM area

Run the assistant locally, then ask document questions.

GitHub Pages can show the interface, but the actual local LLM runs on your computer. Clone the repository, install the dependencies, build the document database, and start asking questions from the terminal or local app.

Run locally conda activate foundry-rag python src/ingest.py python src/answer.py "Where does SQLite store data?"

Pipeline

From files to reliable answers.

01

Read documents

The project loads local TXT, PDF, and DOCX files from the document folder.

02

Create embeddings

Each chunk is converted into a numerical vector using the local embedding model.

03

Retrieve context

The user question is matched with the most relevant document chunks.

04

Answer safely

The local chat model answers only from retrieved context and avoids outside knowledge.
