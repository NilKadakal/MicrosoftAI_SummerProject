Workflow

From file upload to grounded answer.

The project keeps the RAG pipeline visible: ingestion, retrieval, generation, and sources.

01

Upload documents

The local app accepts PDF, DOCX, and TXT files through a simple document upload flow.

02

Index locally

Text is extracted, chunked, embedded, and stored in a local SQLite database.

03

Retrieve context

Questions are matched against document chunks using semantic similarity and filtering.

04

Answer with sources

The assistant answers only from retrieved context and avoids unsupported outside knowledge.

Project overview

A complete local RAG demo, not just a chatbot.

The project demonstrates model setup, embeddings, SQLite storage, retrieval filtering, PDF/DOCX support, automated tests, and a browser-based user experience.

Foundry Local

Runs the chat and embedding models locally on the user's machine during the demo.

SQLite knowledge base

Stores document chunks and embeddings in a lightweight local database file.

Document support

Indexes TXT, PDF, and DOCX files with source labels and retrievable chunks.

Grounded answers

Refuses unrelated questions when the answer cannot be found in the provided documents.
