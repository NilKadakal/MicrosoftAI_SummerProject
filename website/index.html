Architecture

The pipeline is intentionally transparent.

01

Document intake

TXT, PDF, and DOCX files are read from the local document folder.

02

Chunking

Extracted text is split into smaller passages that can be searched reliably.

03

Embeddings

Each chunk is converted into a vector using a local embedding model.

04

SQLite index

Chunks, metadata, and vectors are stored in a lightweight local database.

05

Answer generation

The local chat model answers using only the retrieved context.

Validation

Tested with answerable and unsupported questions.

The project includes automated test output in TEST_RESULTS.md. The goal is not only to answer correct document questions, but also to refuse questions that are not supported by the uploaded files.

Question Expected behavior Status
